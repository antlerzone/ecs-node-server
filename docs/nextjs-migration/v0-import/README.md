# v0 设计导入区（可随时整盘替换）

**这个文件夹专门放从 v0.app 导出的设计代码。你可以每次用新设计整盘替换这个文件夹里的内容，不会影响已经接好后端的前端。**

## 你要做的

1. 在 v0 里改好设计，把生成的代码复制下来。
2. 把代码放进这个文件夹（`v0-import/`）里：
   - 可以按页面建子文件夹，例如 `v0-import/tenant-profile/`、`v0-import/login/`
   - 或者直接放文件，例如 `v0-import/tenant-profile-page.tsx`
   - **每次有新设计就覆盖这里对应的文件或文件夹即可。**
3. 在 Cursor 里对 AI 说（复制下面这句，改一下页面名和路径）：

   > `v0-import` 文件夹里有新的 v0 设计（具体是 xxx 页面），请根据里面的代码更新前端的对应页面（`app/xxx/page.tsx` 或相关 components），只换 UI，保留现有的后端对接（tenant-api、portal-api、contexts、保存/上传逻辑）。

## 前端不会直接「引用」这个文件夹

- **真正跑的前端**是 `app/` 和 `components/`，它们已经接好了 Node 后端。
- **这个文件夹**只是「v0 设计的存放区」：你只管在这里替换/粘贴新设计；由 Cursor（或开发者）**查看这里的代码，把差异合并进** `app/`、`components/`，而不是让 Next 从 v0-import 读代码运行。

这样你就有一个**固定可以整盘替换的地方**，不会误覆盖已接好后端的前端代码。

## 是自动合并吗？

**不是。** 没有脚本或 CI 在监听 v0-import 并自动改 app/。  
流程是：你替换好 v0-import 里的内容后，**在 Cursor 里对 AI 说一句**（例如「v0-import 里 xxx 页面有新设计，请根据里面代码更新前端对应页面，只换 UI，保留后端对接」），由 **AI 根据 v0-import 的代码做一次合并**。每次你要更新前端时，说一句、AI 改一次。
